                              快速开始
================================================================================

方式1: 命令行运行（推荐）
------------------------------------------------------------------------------
1. 测试环境：
   cd ablation_experiments
   python test_env.py

2. 快速实验（验证流程，10-15分钟）：
   python run_ablation_td3.py --episodes 30 --quick

3. 标准实验（论文数据，2-3小时）：
   python run_ablation_td3.py --episodes 200


方式2: Windows批处理文件（双击运行）
------------------------------------------------------------------------------
1. 双击 run_quick_test.bat          → 测试环境
2. 双击 run_ablation_quick.bat      → 快速实验（30轮）
3. 双击 run_ablation_standard.bat   → 标准实验（200轮）


方式3: 单独运行某个配置
------------------------------------------------------------------------------
python run_ablation_td3.py --config No-Cache --episodes 100

可选配置：
  - Full-System        （完整系统-基准）
  - No-Cache          （无缓存）
  - No-Migration      （无迁移）
  - No-Priority       （无优先级）
  - No-Adaptive       （无自适应）
  - No-Collaboration  （无协作）
  - Minimal-System    （最小系统）

================================================================================
                            实验配置说明
================================================================================

7种消融配置：
------------------------------------------------------------------------------
1. Full-System         → 完整系统（所有模块启用）- 作为性能基准
2. No-Cache           → 禁用边缘缓存 - 验证缓存模块影响
3. No-Migration       → 禁用任务迁移 - 验证迁移模块影响
4. No-Priority        → 禁用优先级队列 - 验证调度策略影响
5. No-Adaptive        → 禁用自适应控制 - 验证自适应策略影响
6. No-Collaboration   → 禁用RSU协作 - 验证协作缓存影响
7. Minimal-System     → 最小系统（所有模块禁用）- 整体效果对比

评估指标：
------------------------------------------------------------------------------
主要指标：
  • 平均时延 (avg_delay)           - 越小越好
  • 总能耗 (total_energy)          - 越小越好
  • 任务完成率 (completion_rate)   - 越高越好

辅助指标：
  • 缓存命中率 (cache_hit_rate)
  • 迁移成功率 (migration_success_rate)

================================================================================
                          重要说明和注意事项
================================================================================

✓ 独立环境：
  本文件夹完全独立，所有实验在此进行，不会影响原始项目文件

✓ 算法使用：
  使用原始设计的TD3算法（single_agent/td3.py）

  默认使用随机种子42，确保实验结果可重复

✓ 数据稳定：
  取后50%的Episode数据进行评估，避免初期训练不稳定的影响

✓ 耗时估计：

  - 超快测试（10轮）：5-10分钟
  - 快速实验（30轮）：10-15分钟
  - 标准实验（200轮）：2-3小时
  - 完整实验（500轮）：5-6小时

✓ 资源需求：
  - 存储：每个配置约10-20MB
  - 内存：建议至少8GB RAM
  - GPU：可选，但建议使用以加速训练

================================================================================
                              故障排除
================================================================================

如遇问题：

1. 环境测试失败：
   python test_env.py
   查看具体哪个组件失败

2. 编码问题（emoji显示异常）：
   这是Windows控制台的gbk编码限制，不影响实验运行
   可以使用 test_env.py（无emoji版本）

3. 实验中途中断：
   可以单独运行未完成的配置
   python run_ablation_td3.py --config <配置名> --episodes 200

4. 结果波动大：
   增加训练轮次或使用多个随机种子

5. 内存不足：
   减少batch_size或减少训练轮次
